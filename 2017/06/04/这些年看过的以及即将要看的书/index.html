title: 这些年看过以及即将要看的书

tag: 杂七杂八

回想一下，上一篇blog大概已经是一个月前的事情了，前些天有同学给我说道你的公众号已经很久没更新了，才幡然醒悟，公众号也已经很久没更新了。所以还是要写点东西，之前有同学建议说能不能写点看得懂的东西，可是不写技术类文章不是我的风格（主要我不会写其他的，我也很绝望啊）。哈哈哈，所以就写写这些年看过的书籍吧，顺便给给安利。

Android系列

其实身边好多同学都不解，为什么我要放弃Android，转战后台开发。哈哈哈，至于这个问题嘛，答案可能仅仅是因为纯粹的好玩儿吧。实习嘛，就是挑战自己的机会，选一个自己不是特别熟悉的岗位，也许能学到更多东西嘛。回想学习Android一路走来，也看了很多书籍。纸质书籍看起来有点累，所以除了买了几本高质量的书籍，其余都是看的电子版的。

其实对于Android的，不知道现在还有没有同学想学习这方面的技术，毕竟Google正在研发新的移动端操作系统，天知道啥时候Android就没有未来了。前些天的I/O大会，Google宣布官方支持Kotlin来开发Android应用，相比于Java来说，开发速度更快。不过整体来说，Android开发思路都差不多，要掌握的依然是那些知识点。对于基本的Android开发，UI、四大组件、网络、多线程、消息机制、触摸事件等依然是必须要掌握的知识，更上一层楼的话，进程间通信、热更新、NDK开发等知识可以好好学学。哈，下面是重点推荐。

  • Android入门：第一行代码、第二行代码。郭霖大神的书籍对于基础知识介绍得很全面，也很详细，作为入门书籍来说很合适。
  • Android进阶：Android开发艺术探索，这本书囊括了Android进阶需要掌握的知识，包括进程间通信、View相关的事件体系、Android消息机制等等，作为进阶必备。
  • Android高级：推荐老罗的Android系统源代码情景分析，这本书主要从Android系统的源代码来分析Android的各个核心模块，讲得很详细，由于个人水平有限，这本书仅仅看了部分就放弃了，实在太难了。
  • 杂七杂八：书单里面有写杂七杂八的书其实很不错，比如Android开发必知的50个诀窍，介绍了50个非常有趣并且实用的开发技巧。App研发录同样也从开发应用的角度介绍了开发一个App需要的知识，比如竞品分析、持续集成、热更新等。这两本书都挺有意思的，建议看看。
  • 工具：工欲善其事必先利其器，Android开发使用的IDE从最初的eclipse过渡到现在的Android Studio，开发效率大大增加了，但前提是要熟悉Android Studio这个比较新的工具，所有要好好学习怎么使用它，Android Studio实战以及Android群英传都介绍了怎么高效使用这个先进的武器，值得看看。

没有实践的知识都只能算是纸上谈兵，所以在有了理论基础之后，一定要去开发应用。写第三方微博也好，写天气等小应用都行，利用网上的开放Api，我们可以做很多有意思的事情，只有真正写了应用之后，你才会发现知识的真实性。

Java系列

其实接触的第一门语言并不是Java，但却被Java深深地吸引了（主要是简单）。所以后面开始慢慢自学，最开始从图书馆借书，边看边敲代码，其实最开始根本不懂，为什么就体现出多态性了，为什么通过socket就实现了端到端的通信，为什么就反射了（其实现在也不是特别懂，哈哈哈），当时只知道照着书上的代码敲就好了。于是随着敲的代码越来越多，对Java的熟悉程度也越来越高。

其实对于Java来说，有很多什么21天从入门到精通之类的书籍，当时傻傻的自己也从图书馆借了并看了，但实际上这类书并没多大用。其实对于一门语言的学习，基本语法都差不多，所以随便找一本即可。如果有其他语言的基础，像Java核心卷这类的书籍就很适合用来学Java，如果Java是你第一门语言，那建议先看看Head First Java，因为这本书讲得实在太形象了，它可以让你了解编程可以干嘛，然后再看Java核心技术等来熟悉Java的各种语法。

但对于Java的学习，这只是简简单单的语言学习，而Java真正强大的是其各种方向的框架，比如Android开发，Web开发，大数据等。但这些都建立在对于Java虚拟机熟悉的基础上。如果你对于Java怎么建立对象，怎么分配内存等一无所知，那你还是没有入Java的门。所以《深入理解Java虚拟机》这本书一定要看，并且一定要理解。了解Java的内存模型、了解Java的垃圾回收机制、了解Java的线程安全、了解Java的类加载机制等等，这些都是你入门Java必须要掌握的知识。

然后更上一层，熟悉Java的并发编程的艺术，是成为一个进阶Java工程师必备的知识，因为在实际的开发中，会有很多并发的问题，一旦处理不好，轻则降低系统性能，严重的直接导致系统崩溃。所以学习好Java并发的知识也相当重要。并且现实一点，在面试中，虚拟机和并发是面Java开发工程师必须要面对的问题。

然后Effective Java可以作为Java学习的知识补充，Effective是一个系列，也有c++等，所以感兴趣的话可以读一读。

设计模式作为一种编程思想，对于编程能力的提高是很有帮助的，尤其是面试中如果能够向面试官展示自己对于这方面知识的理解，是有很大的加分的。Head First入门，再合适不过了，然后《设计模式-可复用面向对象软件的基础》很全面，但讲得有点晦涩，请慎重选择。如果对Android开发感兴趣，那么《Android源码设计模式解析与实战》就很合适了，既能熟悉Android知识，又能熟悉设计模式，一举两得。

Java Web系列

这个系列也正是自己目前正在学习的方向，从对于SSM框架（Spring+SpringMVC+Mybatis)的学习，到分布式处理等，需要的只是相当多，也给自己准备了一份书单，争取能够尽快看完、理解并实践。

架构探险和Spring in Action已经看了一遍了，其实本来这篇blog是想将架构探险的读书笔记的，即从零打造一个Java Web框架需要干啥，但写着写着发现自己的理解还是不够深入，所以临时切换成书籍介绍了，下一篇来介绍怎么构建Java Web框架。

可以说Java Web是Java应用的一个很重要的方向，从早期的Java EJB，Java EE到后来的SSH（Struts+Spring+Hibernate)，SSM等框架，Java Web一直在发展。那么对于学习Java Web，我们需要掌握哪些知识呢？

  • Java语言，这毫无疑问，没有Java语言的基础，做Java Web肯定是不行的。
  • 容器，Java Server的容器种类也挺多的，Tomcat、Jetty、Jboss等，不过好像大部分公司包括阿里巴巴等都是用的Tomcat，所以掌握Tomcat的工作原理，性能调优等也是有必要的。
  • 后台服务框架，目前来说，Spring+SpringMVC+ORM的框架应该是主流，Spring是一个很神奇的东西，他为开发者省去了很多事儿，他的DI和AOP特性简直是神器。借助这套框架，我们需要编写Controller来处理相应的url映射，然后编写Service处理业务逻辑，再通过ORM框架将数据持久化到数据库中。目前流行的ORM有Hibernate、Mybatis以及JPA，Spring都对他们提供了支持，所以可以选择任意一种框架来使用。然后中间可能会用到Redis这种缓存框架，不过Spring也提供了对其的支持，用起来也相当简单。
  • 数据库，做后台开发，对数据库的操作也得相当熟悉，传统的关系型数据库比如Oracle、MySQL等总得会用吧，毕竟关系型数据库还是现在的主流。然后文档型数据库MongoDB，key-value数据库Redis等都有其应用场景，也得好好学一学。
  • 刨去业务层的代码，然后就是很多纯技术的东西了，比如怎么去搭建分布式的服务集群来支持更大规模的服务，怎么在有高并发的情况下抵挡对服务器业务及数据的冲击。以及分布式情况下，怎么去平衡CAP（Consistency、Availability和Partition tolerance）之间的关系。怎么去构建服务发现来提高服务的用户体验等等。这部分的知识不仅多，还很难（哭着也要看下去）！

Coding

其实编程是一门综合的艺术，怎么把代码写得好看（当然不是指下面这种），所谓好看的代码是指代码的可阅读性很高，让别人能够一眼看出你这段代码是干啥的。

如果你拿出你三个月前的代码，如果你自己还能一眼就知道某个函数是干啥的，为啥要这样写的话，那说明就是好的代码。对于Coding，一定要有代码规范，对于Java来说，阿里的Java开发规范是一定要去看并遵守的。

哈哈，发现好多面试的书，其实我都没怎么去看。里面重点推荐代码大全、代码整洁之道、阿里Java开发手册，以及重构，改善既有代码的设计这几本书。其实里面都是一些很细节的东西，关键是自己在平时写代码的时候能够注重这些细节，并且遵循这些细节。数学之美是一门很有趣的书，记得当时是在火车上看完的，吴军大神从通信的角度来解决了很多计算机方面的问题，主要是他当初在Google的工作经历，这位大神还有浪潮之巅、文明之光等著作，不过还没来得及拜读，在闲下来的时候读一读对于拓展思维是有很大帮助的。

在看书的路上越走越远

其实很多时候，有同学问我喜不喜欢看书，其实我的答案是否定的，因为我总是不能把读书这个好习惯坚持下去。记得微信读书的书架上摆了好多书，但目前为止也只看完三四本，并且是断断续续看完的。而在阅读这些技术书籍的时候，我貌似已经不把他们当书了，而是一种提升自己实力的工具。所以这些年来也算是看过不少“书”吧，并且在将来还将继续看下去。看着笔记本中存的大量书，瞬间有点心累。

不过对于程序员这个称谓，就意味着要不停地学习，新语言会不停地出现，Kotlin、Go、Ruby等等都是近些年才慢慢发展出来，技术栈也不停地更迭，像之前很火的Android、iOS移动端开发现在也趋于稳定，而之前不温不火的前端到现在又成为大热的技术栈，而人工智能很有可能在不久的将来给IT界带来革命性的改变。所以啊，还是要不停地去学习，不停地看下去吧，希望在全栈的路上能够越走越远，也希望和有志于成为程序员的你共勉！
