Android JNI开发有这一篇就够了

Posted on 2018-06-23   |  

JNI: Java Native Interface，Java本地调用，允许Java和其他语言进行相互调用。常见的是和C及C++进行互相调用。

为什么要用JNI

  • 我们知道C/C++的运行效率比Java更高，在程序中如果希望模块具有更高的运行效率，那么我们应该使用JNI技术。
  • 如果我们希望利用现有的其他语言库，不想在Java中再次实现，那么我们可以利用JNI技术直接调用已实现的库中方法。
  • 如果有一些关键的算法需要隐藏，不希望轻易被别人破解，我们可以用C/C++编写然后编译成so，在Java层用JNI进行调用，提高被破解的成本。
  • 在需要用到和操作系统相关的功能，但Java不具备该功能时，需要利用其他和平台更紧密的语言实现，然后用JNI调用来实现。
  • JVM的初衷是one code, run every，即脱离平台相关性。但引入JNI后，则引入了平台相关性，所以在使用JNI时，在底层语言的实现时往往需要编写多份平台相关性代码。

Get Started

环境准备

  • Android Studio
  • Android SDK
  • Android NDK

一般用最新的Android Studio并下载最新的SDK时，NDK已经包含其中。新建一个工程，在local.properties中都会有相应的配置。

开始编写

创建Java本地方法

类中的方法为如果指定为native则表示这是一个本地方法，当调用该方法时，实际上系统会去相应的地方寻找方法的实现。

创建C头文件

当创建好Java类后，需要创建相应的头文件，利用Java提供的javah命令，我们可以方便地为class文件创建相应的头文件。

首先我们需要将编写的.java文件编译成.class文件，可以利用javac命令，也可以利用Android Studio的编译Build->Rebuild Porject。编译完成后，在工程目录的build->intermediates->classes->debug下，可以找到相应的.class文件，然后执行javah命令即可。

将生成的.h文件拷贝到jni开发目录，我们可以使用Android Studio的功能创建。在main文件夹右键选择->New->New Folder->JNI Folder。

编写C实现

有了C头文件，我们还需要实现里面声明的方法。创建相应的c文件，并实现声明的方法。

编译so文件

编译so文件有两种方式，一是利用Android Studio中的gradle配置，让gradle帮忙编译；另一种方式是自主编写编译脚本，利用ndk-build命令进行编译。

手动编写头文件

可以看出，利用javah指令生成的头文件不仅头文件名字很长，里面的方法名也很长。并且具有一定的规律。

文件名为Java类的全限定名，用_连接。而方法名则为Java_类的全限定名_方法名。其实头文件命名可以自定义，只要保证在Android.mk文件中能够定位到文件路径即可。而方法名之所以这样写，则是为了系统能够顺利加载到相应的方法。

为了可读性，我们可以自定义头文件中的方法，然后在系统加载该so文件时，将Java方法和头文件中的自定义方法进行关联注册即可。

自定义入参和出参

在很多时候，我们的Java层本地方法需要自定义的入参和出参，那么这个Java层面的数据结构在C层如何解析呢。

对于自定义对象，在方法绑定时，我们需要设置正确的入参出参，及遵循上述规则。然后在C代码中，方法的参数为jobject。那么如何从jobject中解析出入参属性以及如何构建出参为jobject呢。

  1. 从jobject中获取类对象
    jclass clz = (*env)->GetObjectClass(env, request);

  2. 从类对象中获取各个属性

  3. 从属性中获取值

通过上述三步，便取出来入参中的属性值。可以看到jni和Java的属性对象有一个映射关系。
对于基本类型

Java Native

在c++中，所有引用类型派生自jobject，而对于c，所有引用类型都用jobject表示。

对于基本类型，jni可以直接强制转换到c/c++中的基本类型，对于应用类型如String，要转换到char *，需要调用JNIEnv的方法进行转换

相反地，构造出参需要jni构造一个jobject，并且此jobject具有出参的类属性。

  1. 通过env的findClass方法得到jclass对象

  2. 获取构造函数，一般获取默认构造函数

  3. 通过构造函数实例化对象

  1. 通过getFieldId获取对象的属性

  2. 通过Set*Field方法为属性赋值，具体方法由数据类型决定，比如SetIntField，SetObjectField等等

通过上述方法，我们便能实现自定义入参和出参了。

JNI的调用流程可以总结如下图


在JVM中完成JNI代理的加载以及方法调用，然后JNI中调用动态链接库，调用真正的本地方法，而本地则执行方法，然后返回结果给到JNI代理，并返回到Java层。

静态注册与动态注册

前面提及到的手动编写头文件与实现文件其实是采用了动态注册的方式，在System.loadLibrary时，会查找JNI_OnLoad函数并执行，因此利用这个机制，我们可以实现方法的动态注册。

而对于静态注册，及方法名超级长的那种方法，除了方法名长的缺点外，程序在第一次执行方法时，会根据方法名去搜索对应JNI层的函数，然后建立关联，效率比动态注册更低。
