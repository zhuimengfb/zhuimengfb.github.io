利用Alfred Workflow批量签名Android App

Posted on 2016-10-07   |  

利用Android Studio打包完成App的迭代开发，准备发布到应用市场，上传apk文件后，出现您的安装包存在危险，建议进行加固处理，顿时傻眼。好吧，按照提示进行加固处理，加固完后下载经过加固处理的apk，提示需要进行重新签名，泪崩。好吧，言归正传，这篇文章教你如何高效地进行Android apk重新签名。

Geek方法

貌似高手一般都用命令行搞定一切，用GUI好像显得逼格不够。好吧，其实重新签名最原始的方式就是命令行。使用Java自带的工具可以方便地进行签名。方法比较简单，直接贴命令了。

jarsigner -verbose -keystore yourkeystorepath -storepass "your keystore password" -keypass "your private key password" -signedjar signedapkpath sourceapkpath aliasname

命令就这样，只需要替换

  • yourkeystorepath(你的密钥的绝对位置)
  • your keystore password(你的签名密码)
  • your private key password(你的私钥密码)
  • signedapkpath(重新签名后的apk绝对路径)
  • sourceapkpath(需要重新签名的apk的绝对路径)
  • aliasname(密钥的别名)

执行这个命令，重新签名就完事儿了。

懒人方法

好吧，个人还是比较懒，每次都懒得去敲这么长的命令，尤其是各种路径。windows下貌似有很多GUI工具来进行签名，mac下貌似就没有这种好事儿了。不过我们有Alfred神器啊，自从有了Alfred加各种Workflow，简直不能更开心，其他东西就不说了，说说今天的正事儿：使用Alfred Workflow批量签名Android apk，网上有人贴出了相关方法，但我试了一下没有成功，然后就在前人的基础上进行学习研究，学了一点bash的相关脚本语法，最后，我就直接说步骤吧。

  1. 打开Alfred->preference，点击Workflows，点击左下角加号，选择Blank Wrokflow，进入设置界面，简单设置名称等等，如下图所示
  2. 新建完成后，在右侧空白区域双击，在弹出菜单中选择Triggers->File Action，然后又随便明个名字，把Accepts multiple files勾选上。

  3. 编写签名脚本，这是最为核心的一步，依然是在空白地方双击，选择Actions->Run Scripts,然后弹出一个编写脚本的界面。语言选择bash，输入选择query方式，运行方式选择顺序执行，然后把一下代码复制到脚本框中

    in="{query}"

blah='    ' read -a array <<< "$in"

suc_count=0
fail_count=0
for i in "${array[@]}"
do
E=$(jarsigner -verbose -keystore yourkeystorepath  -storepass "your keystore password" -keypass "your private key password" -signedjar ${i/.apk/_signed.apk} $i alias)

if [ ${E:2:7} = 'adding' ] || [ ${E:1:8} = 'updating' ]
    then
    suc_count=$(($suc_count+1))

    else
        fail_count=$(($fail_count+1))
fi

done

echo "Success: $suc_count"
echo "Fail: $fail_count"

    上述中只需要换成你自己的keystore相关的配置即可，即位置，相关密钥和别名，其他不需要改动，签名成功后会自动在需要签名的apk目录下生成签名成功的apk。完整的设置如下图

  4. 设置通知，我们总不希望脚本执行完了什么通知也没有吧？依然空白地方双击，选择Outputs->Post Notification，进行如下图所示设置
  5. 把三个模块进行连接，便完成了Workflow的编写，如下图所示
  6. 好了，可以试一试使用了，选中需要重新签名的apk，可以多选，然后按⌥⌘ \(alt+commd+\),会唤醒触发器，下面边有一个我们的workflow，选择使用即可，最后静静等待重新签名完成吧。

因为前人的脚本并不正确，一行一行进行测试的确有点烦，好在最后搞定了，为以后重新签名省了不少时间，果然人懒就会在其他方面想办法（投机取巧）。
