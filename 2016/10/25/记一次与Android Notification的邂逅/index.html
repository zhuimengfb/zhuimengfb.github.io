记一次与Android Notification的邂逅

Posted on 2016-10-25   |  

项目中最近做到了聊天功能，那么消息通知功能是逃不掉的，当有新的消息到达时，我们希望程序能够通知用户有新消息到达。在Android中，消息通知这一功能主要由Notification来完成，根据Google的官方解释，Notification是您可以在应用的常规 UI 外部向用户显示的消息。当您告知系统发出通知时，它将先以图标的形式显示在通知区域中。用户可以打开抽屉式通知栏查看通知的详细信息。 通知区域和抽屉式通知栏均是由系统控制的区域，用户可以随时查看。

创建简单通知

一个简单的通知主要包含了应用图标，通知标题，通知内容和通知时间构成，如下图所示，因此我们在创建通知时，这几项一般都是必选项。

创建通知对象的方式采用了Builder模式，为了提高不同Android版本的兼容性，我们使用v4包中的NotificationCompt对象。

  1. 创建NotificationCompat.Builder对象

    NotificationCompat.Builder builder = new NotificationCompat.Builder(this);

  2. 设置必选内容

    builder.setSmallIcon(R.mipmap.ic_launcher);
builder.setContentTitle("这是通知标题");
builder.setContentText("这是通知内容");

    除了这些，还可以加上其他内容，比如时间

    builder.setWhen(System.currentTimeMillis());

    其他的请参考NotificationCompat.Builder Api

  3. 获取系统的通知管理器NotificationManager

    NotificationManager manager = (NotificationManager) getSystemService(Context.NOTIFICATION_SERVICE);

  4. 通过通知管理器的notify方法弹出通知

    int notificationId = 0;
manager.notify(notificationId, builder.build());

    其中，notify方法的第一个参数是用来标识该通知的id，通过这个id，我们可以手动cancel该通知。使用方法如下

    manager.cancel(notificationId);

    至此，创建一个简单的通知应该没问题了。

聊天中的消息通知系统

对于聊天应用中的消息通知，我们需要考虑更多东西，这次方案设计的思路是这样的。

  1. 当model层收到消息时，即发送一条通知。
  2. 然而考虑到用户有可能正在聊天界面，这时候就不应该弹出通知打扰用户体验。
  3. 但是用户有可能正在和多人聊天，比如当前界面是处于和A聊天，但此时B发来一条消息，我们还是应该通知用户有新消息到达。
  4. 通知栏中可以有多条通知，一个聊天对象的通知应该合并在一起显示
  5. 点击通知，应该跳转到相应的聊天界面，并且点击多条通知后，聊天界面依然只能有一个。
  6. 从其他地方进入聊天界面应当自动消除通知栏消息

这差不多是最简单的流程了，由于项目中暂时还没有添加消息免打扰等功能，所以暂时不考虑不同聊天对象的通知策略。接下来就顺着这个流程来实现聊天中的消息通知。

对于第一条，实现比较简单，由于创建通知我们只需要Context对象，因此我们可以使用应用的全局Context来实现，因此，通知系统完全可以独立出来，不依赖任何Activity或者Service，所以我创建了一个NotificationUtils类。然后在该类中写了一个showNotification方法，传入聊天信息这个对象。这样只需要收到消息的时候，直接调用该方法并传入聊天消息即可。此时代码如下所示

public class NotificationUtils {

  public static void showNotification(ChatMessage chatMessage) {
    NotificationCompat.Builder builder = new NotificationCompat.Builder(App.getContext());
    builder.setSmallIcon(chatMessage.getSenderAvatar());
    builder.setContentTitle(chatMessage.getSenderName());
    builder.setContentText(chatMessage.getContent());
    builder.setWhen(chatMessage.getCreateTime());
    NotificationManager manager = (NotificationManager) App.getContext().getSystemService(Context
        .NOTIFICATION_SERVICE);
    int notificationId = 0;
    manager.notify(notificationId, builder.build());
  }
}

然而考虑到第二、三条，当我们处于和A聊天时，这时候A发送的消息不应该通知，而其他用户发送的通知则需要通知。为了实现这个，也是灵机一动。由于在聊天界面，我们肯定会持有当前聊天对象的信息，比如Id，而发送的ChatMessage肯定也会携带发送者的Id，那么对比这两个Id，我们则可以判断是否需要显示通知。这时候建一个全局类来记录当前聊天对象的Id，当用户进入聊天界面时，更新该Id，当用户退出聊天界面时，将Id设置为空。这样在通知的工具类中，我们只需要加一个判断即可。

if (TextUtils.equals(GlobalConfig.getCurrentChatTargetId(),chatMessage.getTargetId())){
  //do not show notification
} else {
  //show notification
}

然后对于第四条，这时候notificationId便派上用场了，不同的notificationId会显示成不同的notification，相同的notificationId的通知会自动合并。因此问题得核心转变为为每个聊天对象生成不同的notificationId。这里也灵机一动，考虑到每条ChatMessage中会带有聊天对象的id，那么我们可以在NotificationUtils这个类中持有一个映射表，每个id映射成不同的整型数字即可。

private static Map<String, Integer> idMap = new HashMap<>();

  private static int getNotificationId(String targetId) {
    Integer id = idMap.get(targetId);
    if (id == null) {
      idMap.put(targetId, idMap.size());
    }
    return idMap.get(targetId);
  }

这样在notify中，我们只需要调用上面的get方法得到notificationId即可。

最后，便是通知点击跳转。不用说，跳转肯定需要Intent来实现。在Intent中设置好跳转的Activity。然后创建PendingIntent对象，将该对象传递给notification的setContentIntent方法。代码如下

Intent intent = new Intent();
   intent.setClass(App.getContext(), ChatroomActivity.class);
PendingIntent pendingIntent = PendingIntent.getActivity(App.getContext(), notificationId,
      intent, PendingIntent.FLAG_CANCEL_CURRENT);
builder.setContentIntent(pendingIntent);

这样写完，实际测下来点击跳转没啥问题，然而会产生多个聊天界面，这是我们不想看到的。这时候Intent的flag就起作用了，我们只需要加上这么一句

intent.setFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP | Intent.FLAG_ACTIVITY_NEW_TASK);

第一个标志保证了如果匹配到相应的Activity会清除掉匹配Activity之上的所有Activity，第二个标志保证了新的Activity会是一个全新创建的Activity。这样便保证了点击不同的notification会进入到不同的聊天界面，并且有且只有当前一个聊天界面。

对于最后一点，由于NotificationUtils维护了一张映射表，我们只需要在进入聊天界面Activity的onCreate方法中通过聊天对象的id来查找到notificationId，并且调用cancel方法即可。

这样一个简单的聊天的消息通知系统便做好了，之后便是加入各个聊天对象的策略模式了，比如响铃、震动、静音、通知栏不显示等等。

其实通知栏还有很多高级用法，比如常驻通知栏、比如直接通知栏展开显示更多消息，比如通知栏直接进行回复消息等操作。这些高级属性等下次使用时再来更新。。
