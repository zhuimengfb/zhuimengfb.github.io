设计模式之独爱第三者迭代器Iterator

Posted on 2016-10-11   |  

在Java，或者其他语言中，集合的遍历总是会经常要用到，在学数据结构的时候，我们往往会自己去实现数据遍历过程，但在面向对象编程的语言中，我们只希望集合仅仅维护自己的数据就行了，遍历操作应该提取出来。所以最简单的方式就是由客户端自己写一段遍历代码去遍历集合的数据，然而当很多集合需要进行遍历操作时，我们只能一遍又一遍地重复着for循环，这肯定是无法让人接受的。因此，我们大爱的第三者迭代器诞生了。

迭代器模式是设计模式中的一种行为模式，主要用来对List,Map,Set等集合进行数据遍历。迭代器模式在对外提供了结合的遍历接口之外，还隐藏了集合的数据结构。如果我们使用for循环来遍历集合，我们得知道集合的大小才行，这显然不符合封装的特性。而迭代器模式加上泛型的结构，我们能够很方便地对各种集合进行遍历访问。
迭代器模式的UML图如下所示

集合基类可以是接口也可以使抽象类，里面定义了建立迭代器的抽象方法，在具体的集合类中，实现或者继承基类并且实现抽象方法。而迭代器基类中定义hasNext(),next()等抽象方法，具体类中实现这些方法来对集合进行遍历。下面来简单写个例子吧。

  • 集合基类

    public abstract class Aggravate {
    abstract Iterator createIterator();
}

  • 迭代器基类

    public interface Iterator<T> {

    boolean hasNext();

    T next();
}

  • 具体集合类

    public class ConcreteIterator<T> implements Iterator<T> {

    private List<T> dataList;
    private int currentIndex = 0;

    private ConcreteIterator() {
    }

    public ConcreteIterator(List<T> data) {
        this.dataList = data;
    }

    @Override
    public boolean hasNext() {
        return currentIndex < dataList.size();
    }

    @Override
    public T next() {
        return dataList.get(currentIndex++);
    }
}

  • 具体迭代器类

    public class ConcreteAggravate extends Aggravate{

    private List<String> dataList;
    private ConcreteIterator<String> concreteIterator;

    public ConcreteAggravate() {
        dataList = new ArrayList<>();
    }

    public void add(String data){
        dataList.add(data);
    }

    @Override
    Iterator createIterator() {
        return new ConcreteIterator(dataList);
    }
}

  • 测试代码

    public static void main(String[] args) {
    ConcreteAggravate concreteAggravate = new ConcreteAggravate();
    concreteAggravate.add("123");
    concreteAggravate.add("456");
    concreteAggravate.add("789");
    concreteAggravate.add("012");
    Iterator iterator = concreteAggravate.createIterator();
    while (iterator.hasNext()) {
        System.out.println(iterator.next());
    }
}

    运行程序可以得到正确的打印。由于迭代器模式大大简化了客户端对集合的遍历，因此很多语言都内置了集合的迭代器，我们可以直接使用即可。同样，在Android中，我们对数据的查询操作一把不会返回具体的数据，而是返回一个游标Cursor，这个Cursor便是Android中迭代器模式的范例，通过Cursor，我们可以一行一行地拿到查询出的数据，并且转换成我们需要的JavaBean。迭代器模式比较简单，虽然其增加了代码量，增加了类的数量，但其方便了客户端的遍历操作，并且对客户端隐藏了集合的具体实现，因此迭代器模式在多种语言中都运用广泛。
