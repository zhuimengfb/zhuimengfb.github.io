Xposed入门实践

Posted on 2016-11-23   |  

最近在看Android系统层的东西，突然想起了之前刷机时经常玩的Xposed框架，心血来潮想要自己试一试Xposed模块开发，然后查阅了一些资料，上手试了试，简答做一下总结。

前期准备

Xposed框架的运行必须是在root了的手机上，对于经常玩机的人来说，root一部手机基本上是手到擒来。但对于不想root手机的人怎么办呢，也有办法，那就是用Genymotion提供的虚拟机，Genymotion的虚拟机基本都是root了的，并且Genymotion的虚拟机运行性能也还不错，所以推荐大家装一下Genymotion。过程大致如下

  1. Genymotion官网以个人用户形式注册
  2. 下载安装Genymotion
  3. 如果安装完之后直接运行报错的话，一般都是没有装VirtualBox，那么去装一个就好了。
  4. 装完之后应该运行就不会出错了，选择虚拟机设备的型号以及Android版本，创建一个虚拟机就可以运行虚拟机了。
  5. 新版本的Genymotion貌似不用和Android Studio进行绑定了，我只是设置了Genymotion中的SDK路径，然后Android Studio中装的Genymotion plugin就可以搜索到创建的虚拟机了。


然后在虚拟机中装好Xposed框架，注意选择Xposed Framework的时候选择x86的版本，选择好了之后，直接将zip包拖进虚拟机就可以安装好重启即可，装好后是这个样子的。

虽然有一串红字，但不影响Xposed框架的运行。前期准备工作就完了。

Android Studio Xposed模块开发

首先是引入依赖包，对于Android Studio来说就十分方便，只需要在app的build.gradle文件中加入以下依赖即可

provided 'de.robv.android.xposed:api:82'

注意这里是provided而不是我们平时使用的compile。然后在Manifest文件中对Xposed模块进行相应的描述

<meta-data
    android:name="xposedmodule"
    android:value="true" />
<meta-data
    android:name="xposeddescription"
    android:value="Easy example which makes the status bar clock red and adds a smiley" />
<meta-data
    android:name="xposedminversion"
    android:value="82" />

这些标签都放在application标签下。然后便是最重要的Hook类开发了。在包下新建一个类，并且实现IXposedHookLoadPackage接口，这时要求实现handleLoadPackage方法。最简单的一个Hook类如下。

public class XposedTest implements IXposedHookLoadPackage {
      @Override
      public void handleLoadPackage(XC_LoadPackage.LoadPackageParam loadPackageParam) throws Throwable {
        XposedBridge.log("Loaded app: " + loadPackageParam.packageName);
      }
}

Xposed框架的运行原理便是在其他应用进行加载的时候会自动执行handleLoadPackage方法加载软件。因此该类在启动手机后会自动加载开机自启的软件的包名信息，打印到Xposed的log日志中。

还有一部比较关键的是为系统指定Xposed运行入口，我们编写实现了IXposedHookLoadPackage接口的类后，Xposed框架并不知道需要执行的类入口在哪儿，因此我们需要assets中添加一个名为xposed_init的文件，里面写入我们写的类的路径。

com.example.fbi.wxtest.xposed.XposedTest

写完之后便可以安装到虚拟机中运行，这时候Xposed应用中便会出现我们写的模块，勾选后重启手机便会生效。

Xposed之所以强大是因为Xposed框架基本能够hook任意的函数。我们只需要应用逆向工程的方法去找到第三方app的方法，然后我们就可以随意改变该方法的代码，从而影响第三方应用的执行效果。具体怎么hook方法，且听下回分解。
