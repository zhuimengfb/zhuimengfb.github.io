Android Studio 生成单例模板

Posted on 2016-09-24   |  

最近在写有关SharedPreferences的代码，由于整个工程中，对于SharedPreferences的操作无非是存储，读取，所以将其封装成了一个工具类。工具类一般都是静态类，但对于SharedPreferences工具的使用率并不会像其他工具那么频繁，所以希望用懒加载的模式，当需要的时候才生成实例，所以可以将该类设计为单例模式。由于单例模式的写法都差不多，不想每次都写重复的代码，所以利用Android Studio的模板功能简单写一个模板用来生成单例模式代码。

Android Studio的模板相关设置位置再Settings->Editor->Live template，然后便可以看到Android Studio的众多模板代码。我们选择Android，然后点击旁边的+选择新增模板。

首先填写模板触发的简写词组以及模板简介

然后编写模板代码

private static final String KEY_PREFERENCES = "$key_preferences$";
private static $class$ $variable$;
private static SharedPreferences sharedPreferences;

private $class$(){}

public static $class$ getInstance(Context context){
    if ($variable$ == null){
        synchronized($class$.class){
            if ($variable$ == null){
              $variable$ = new $class$();
              sharedPreferences= context.getSharedPreferences(KEY_PREFERENCES,Context.MODE_PRIVATE);
            }
        }
    }
    return $variable$;
}

其中class为该类的类名，variable为单例变量名，这里选择了较为简单的双重校验锁方式。sharedPreferences为我们需要操作的SharedPreferences类，然后需要定义该sharedPreferences的文件名称。

为了不需要让我们自己填写类名，我们可以设置class的默认值为该类类名。

点击添加变量，如下图所示

最后，选择该模板的使用范围，我们一般选择在生命变量的时候使用，点击左下角的使用范围，最终如下图所示

使用的时候直接在生命单例变量的时候输入模板简写即可。
