Android Training之数据保存

Posted on 2016-08-01   |  

在Android系统中，保存数据主要有三种方式：

  1. 轻量级数据保存方案：SharedPreferences，主要以键值对的方式保存，一般用来保存应用的设置，用户等轻量级信息。
  2. 文件保存方案：重量级数据可以通过文件的方式存储到系统的内置存储区域或者外置存储区域。
  3. 结构性数据保存方案：对于应用的结构化数据，可以使用数据库的方式来进行存储，在Android中主要使用SQLite来进行数据库存储。

SharedPreferences保存键值对数据

Android提供了轻量级的数据存储方案，即SharedPreferences。该方案的文件一般存储在系统根目录的/data/data/app_package_name/shared_prefs/目录下，文件格式为xml文件，其中数据都保存在map根标签下。
在Activity或者Fragment中，获取SharedPreferences的方式主要有两种。第一种方式是通过Context类的getSharedPreferences方法去获取，另一种方式是通过Context类的getPreferences方法来获得。

  1. getSharedPreferences(String name, int mode)，name为共享文件的名字，在第一次调用该函数的时候，如果没有相应名字的文件，将会新建共享文件并返回共享文件的句柄。mode为共享文件的共享私密性，分别可以设置为Context.MODE_PRIVATE(本应用可读写)，Context.MODE_WORLD_READABLE(其他应用只读)，Context.MODE_WORLD_WRITEABLE(其他应用读写)。

  2. getPreferences(int mode)，使用该方法得到的SharedPreferences会和Activity绑定，即会以这个Activity的类名作为name，如果该类的包名和应用包名不同，则使用该Activity的全称作为name，mode和上述一致。翻看源码，实际上该方式最终还是使用的第一种方式创建，只是自动传递了name进去。

    public SharedPreferences getPreferences(int mode) {
    return getSharedPreferences(getLocalClassName(), mode);
}

public String getLocalClassName() {
    final String pkg = getPackageName();
    final String cls = mComponent.getClassName();
    int packageLen = pkg.length();
    if (!cls.startsWith(pkg) || cls.length() <= packageLen
        || cls.charAt(packageLen) != '.') {
        return cls;
    }
    return cls.substring(packageLen+1);
}

当获取到可以读写的SharedPreferences实例后，就可以进行数据保存以及读取了。数据获取方式比较简单，通过getXxx(String key, Xxx default)方式来获取各种类型数据。当该key值的数据不存在时，将获得default数据。而数据存储则要先通过SharedPreferences实例的edit()方法获得editor实例，然后通过各种putXxx(String key, Xxx value)方法存储键值对数据，最后通过commit()或者apply()方法进行保存。commit()方法会立即保存数据，而apply()方法则是异步，在合适的时间保存，如果对UI的流畅度要求较高，建议使用apply()方法保存。

实际项目中，一个SharedPreferences可能会在多个Activity中用到，比如用户的简单用户信息，这时候可以将SharedPreferences写成公共的工具类方便不同的Activity进行调用。比如下面的例子。

public class UserSharedPreferencesUtil {

      private static final String USER_SHARED_PREFERENCES_NAME = "UserInfo";
      private static final String USER_SHARED_PREFERENCES_KEY_USER_NAME = "user_name";
      private static SharedPreferences sharedPreferences;
      private static UserSharedPreferencesUtil userSharedPreferencesUtil;

      private UserSharedPreferencesUtil() {
      }

      public static UserSharedPreferencesUtil getInstance(Context context) {
        if (userSharedPreferencesUtil == null) {
              synchronized (UserSharedPreferencesUtil.class) {
                if (userSharedPreferencesUtil == null) {
                      userSharedPreferencesUtil = new UserSharedPreferencesUtil();
                      sharedPreferences = context.getSharedPreferences(USER_SHARED_PREFERENCES_NAME, Context
          .MODE_PRIVATE);
                }
              }
        }
        return userSharedPreferencesUtil;
      }

      public void saveUserName(String userName) {
        sharedPreferences.edit().putString(USER_SHARED_PREFERENCES_KEY_USER_NAME, userName).apply();
      }

      public void getUserName(){
        sharedPreferences.getString(USER_SHARED_PREFERENCES_KEY_USER_NAME, "");
      }
}

保存文件

在Android系统中，文件存储位置主要在内部存储和外部存储两种存储介质。一般内部存储空间有限，存储在内部存储的文件一般只有该应用可读写，并且会随着应用的卸载而清空。而外部存储的文件则会永远保存，直到用户手动删除，并且外部空间可以扩展，早期的Android手机一般会搭载移动SD卡来扩展外部存储。至于文件本身的操作，则和Java中的文件操作基本一致，即操纵File类以及输入输出流等类来操纵文件的读写。

在Android系统中，要操作文件，需要一些额外的步骤：

  1. 申请读写手机存储空间的权限。在Android6.0之前，只需要在Manifest文件中进行权限申明即可，如下所示，如果只想申请读文件权限，则可只使用READ_EXTERNAL_STORAGE权限。

    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

    而在Android6.0之后，引入的动态权限规则需要用户动态授予读写文件的权限，开发者需要在进行文件读写之前动态检查读写权限，如果应用未被授权，则需要重新申请权限，并且在回调中处理授予权限的结果。简单代码如下所示。

    if (ActivityCompat.checkSelfPermission(this, Manifest.permission.WRITE_EXTERNAL_STORAGE) !=
PackageManager.PERMISSION_GRANTED) {
      ActivityCompat.requestPermissions(this, new String[]{Manifest.permission
  .WRITE_EXTERNAL_STORAGE}, REQUEST_WRITE_EXTERNAL_STORAGE_CODE);
}

@Override
  public void onRequestPermissionsResult(int requestCode, @NonNull String[] permissions, @NonNull int[] grantResults) {
    super.onRequestPermissionsResult(requestCode, permissions, grantResults);
    if (requestCode == REQUEST_WRITE_EXTERNAL_STORAGE_CODE) {
          if (grantResults[0] == PackageManager.PERMISSION_GRANTED) {
            //TODO permission granted do something
          } else {
            //TODO permission denied do something
          }
    }
  }
  2. 保存文件到内部存储中，可以使用Context类的getFileDir()方法或者getCacheDir()方法来获取应用的文件目录。其中的区别是，当系统存储空间不足时，cacheDir中的文件会被删除。打开文件的方式既可以使用Java中的File类，也可以使用Android提供的openFileOutput(String name, int mode)方法获取输出流或者openFileInput(String name)获取输入流。
  3. 保存文件到外部存储中，可以使用Environemnt类的相关方法直接操作整个外部存储或者Context类的相关方法保存文件到该应用的外部目录中。比较常用的是用getExternalStoragePath()方法获得外部存储根目录的File类，getExternalStoragePublicDIrectory(int type)获取外部存储的公用文件目录，如Music，Movie等，以及getExternalStorageState(）方法获得外部存储的各种状态。Context类可以使用getExternalFilesDir()或者getExternalCacheDir()等方法获得和应用相关的专有文件目录。
  4. 文件删除，当确保一个文件永远不会用到时，应该立即删除该文件，可以通过File类的delete()或者Context类的deleteFile()方法。

在应用被卸载时，内部存储文件和getExternalFileDir()中的文件都会被自动清除。

数据库存储结构性数据

为了方便存储结构性数据，Android内置了SQLite数据库。直接上操作步骤吧。

  1. 创建Entry类，一般结构性数据都是JavaBean，创建JavaBean的属性和数据库列的关联，即创建单独的Entry类，里面定义了各种共有的属性。实现的BaseColumns接口中定义了_ID属性，可以直接使用。

    public class UserEntry implements BaseColumns {

      public static final String TABLE_NAME = "tb_user_info";
      public static final String ENTRY_USER_ID = "userId";
      public static final String ENTRY_USER_NAME = "userName";
      //more
}

  2. 创建自定义的SQLiteOpenHelper类，在该类中，执行数据库的创建以及升级操作。

    public class UserSQLiteHelper extends SQLiteOpenHelper {

      public static final String DB_NAME = "user.db";
      public static final int DB_VERSION = 1;

      public UserSQLiteHelper(Context context) {
        super(context, DB_NAME, null, DB_VERSION);
      }

      @Override
      public void onCreate(SQLiteDatabase db) {
        db.execSQL("CREATE TABLE " + UserEntry.TABLE_NAME + " (" + UserEntry._ID + " INTEGER PRIMARY " +
"KEY, " + UserEntry.ENTRY_USER_ID + " VARCHAR, " + UserEntry.ENTRY_USER_NAME + " VARCHAR)");
      }

      @Override
      public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        db.execSQL("DROP TABLE IF EXISTS " + UserEntry.TABLE_NAME);
        onCreate(db);
      }
}

  3. 创建自定义DataHelper类进行数据库的增删改查等操作。首先通过自定义的SQLiteOpenHelper类获取可读写的SQLiteDataBase实例，然后通过该实例就可以执行数据库的增删改查。操作有两种方式，一种是通过传统的SQL语句方式，另一种则是Android封装好的方法，具体看如下代码。

    public class UserDataHelper {

      private static SQLiteDatabase database;
      private static UserDataHelper userDataHelper;

      private UserDataHelper() {
      }

      public static UserDataHelper getInstance(Context context) {
        if (userDataHelper == null) {
              synchronized (UserDataHelper.class) {
                if (userDataHelper == null) {
                      userDataHelper = new UserDataHelper();
                      UserSQLiteHelper userSQLiteHelper = new UserSQLiteHelper(context);
                      database = userSQLiteHelper.getWritableDatabase();
                }
              }
        }
        return userDataHelper;
      }

      public void insertUser(UserInfo userInfo) {
        ContentValues contentValues = new ContentValues();
        contentValues.put(UserEntry.ENTRY_USER_ID, userInfo.getUserId());
        contentValues.put(UserEntry.ENTRY_USER_NAME, userInfo.getUserName());
        database.insert(UserEntry.TABLE_NAME, null, contentValues);
      }

      public UserInfo queryUserInfo(String userId) {
        Cursor cursor = database.query(UserEntry.TABLE_NAME, new String[]{UserEntry.ENTRY_USER_ID,UserEntry.ENTRY_USER_NAME}, UserEntry.ENTRY_USER_ID + "=?", new String[]{userId}, null,null, null);
        UserInfo userInfo = null;
        if (cursor.moveToFirst()) {
              userInfo = new UserInfo();
            userInfo.setUserId(cursor.getString(cursor.getColumnIndex(UserEntry.ENTRY_USER_ID)));
              userInfo.setUserName(cursor.getString(cursor.getColumnIndex(UserEntry.ENTRY_USER_NAME)));
        }
        return userInfo;
     }

      public void deleteUserInfo(UserInfo userInfo) {
        database.delete(UserEntry.TABLE_NAME, UserEntry.ENTRY_USER_ID + "=?", new String[]{userInfo.getUserId()});
      }

      public void updateUserInfo(UserInfo userInfo) {
        ContentValues contentValues = new ContentValues();
        contentValues.put(UserEntry.ENTRY_USER_ID, userInfo.getUserId());
        contentValues.put(UserEntry.ENTRY_USER_NAME, userInfo.getUserName());
        database.update(UserEntry.TABLE_NAME, contentValues, UserEntry.ENTRY_USER_ID + "=?", new String[]{userInfo.getUserId()});
      }

      public void insertUserInfoBySql(UserInfo userInfo) {
        database.execSQL("insert into " + UserEntry.TABLE_NAME + "(" + UserEntry.ENTRY_USER_ID + ","+ UserEntry.ENTRY_USER_NAME + ") values " + "('" + userInfo.getUserId() + "','" + userInfo.getUserName() + "')");
    }

}

    实例代码中，主要使用了Android提供的insert，query，update以及delete方法来实现增删该查，其中这些方法还有很多重载方法实现更高级的功能，这里就不赘述了。如果使用传统的SQL语句来实现增删该查功能，写SQL语句时一定要注意其中的空格，引号等，稍有不慎就会出错。此外，还可以使用事务来保证数据库的可靠执行，使用方式是beginTransaction和endTransaction方法。
